033package org.naomi.regex;
034
035/**
036 * 
037   This class (which is analogous to but distinct from {@link
038   java.util.regex.Matcher java.util.regex.Matcher}) is the 
039   <a href="./package-info.html"> Naomi package's </a>
040   engine that performs match operations on a {@link java.lang.CharSequence
041   </code>character sequence<code>} by interpreting a {@link Pattern}.
042 
043 <p>
044
045  A Matcher is created from a {@link Pattern} by invoking the Pattern's {@link
046  Pattern#matcher matcher} method.  Once created, a Matcher can be used to
047  perform three different kinds of match operations:
048 
049  <ul>
050 
051    <li><p> The {@link #matches matches} method attempts to match the entire
052    input sequence against the pattern.  </p></li>
053 
054  <li><p>
055
056    The {@link #lookingAt lookingAt} method attempts to match the input
057    sequence, starting at the beginning of the {@link #region region}, against
058    the pattern.
059
060  </p></li>
061 
062    <li><p> The {@link #find find} method scans the input sequence looking for
063    the next subsequence that matches the pattern.  </p></li>
064 
065  </ul>
066 
067  <p> Each of these methods returns a boolean indicating success or failure.
068  More information about a successful match can be obtained by querying the
069  state of the matcher.
070 
071  <p> A matcher finds matches in a subset of its input called the
072  <i>region</i>. By default, the region contains all of the matcher's input.
073  The region can be modified via the {@link #region region} method and queried
074  via the {@link #regionStart regionStart} and {@link #regionEnd regionEnd}
075  methods. The way that the region boundaries interact with some pattern
076  constructs can be changed. See {@link #useAnchoringBounds
077  useAnchoringBounds} and {@link #useTransparentBounds useTransparentBounds}
078  for more details.
079 
080  <p> This class also defines methods for replacing matched subsequences with
081  new strings whose contents can, if desired, be computed from the match
082  result.  The {@link #appendReplacement appendReplacement} and {@link
083  #appendTail appendTail} methods can be used in tandem in order to collect
084  the result into an existing string buffer, or the more convenient {@link
085  #replaceAll replaceAll} method can be used to create a string in which every
086  matching subsequence in the input sequence is replaced.
087 
088  <p> The explicit state of a matcher includes the start and end indices of
089  the most recent successful match.  It also includes the start and end
090  indices of the input subsequence captured by each <a
091  href="Pattern.html#cg">capturing group</a> in the pattern as well as a total
092  count of such subsequences.  As a convenience, methods are also provided for
093  returning these captured subsequences in string form.
094 
095  <p> The explicit state of a matcher is initially undefined; attempting to
096  query any part of it before a successful match will cause an {@link
097  IllegalStateException} to be thrown.  The explicit state of a matcher is
098  recomputed by every match operation.
099 
100  <p> The implicit state of a matcher includes the input character sequence as
101  well as the <i>append position</i>, which is initially zero and is updated
102  by the {@link #appendReplacement appendReplacement} method.
103 
104  <p> A matcher may be reset explicitly by invoking its {@link #reset()}
105  method or, if a new input sequence is desired, its {@link
106  #reset(java.lang.CharSequence) reset(CharSequence)} method.  Resetting a
107  matcher discards its explicit state information and sets the append position
108  to zero.
109 
110  <p>
111   Instances of this class are not safe for use by multiple concurrent threads.
112
113</p>
114 *
115 * @author      Norman Shapiro
116 * @author      Jeff Rothenberg
117 */
118

337    /**
338     * Implements a non-terminal append-and-replace step.
339     *
340     * <p> This method performs the following actions: </p>
341     *
342     * <ol>
343     *
344     *   <li><p> It reads characters from the input sequence, starting at the
345     *   append position, and appends them to the given string buffer.  It
346     *   stops after reading the last character preceding the previous match,
347     *   that is, the character at index {@link
348     *   #start()}&nbsp;<tt>-</tt>&nbsp;<tt>1</tt>.  </p></li>
349     *
350     *   <li><p> It appends the given replacement string to the string buffer.
351     *   </p></li>
352     *
353     *   <li><p> It sets the append position of this matcher to the index of
354     *   the last character matched, plus one, that is, to {@link #end()}.
355     *   </p></li>
356     *
357     * </ol>
358     *
359     *
360     * <p> This method is intended to be used in a loop together with the
361     * {@link #appendTail appendTail} and {@link #find find} methods.  The
362     * following code, for example, writes <tt>I say that $17 + $9 = $26
363     *  everyday.
364     *  </tt> to the standard-output stream: </p>
365     *
366     * <blockquote><pre>
367     Pattern digits=new BuiltInPattern(CoreBuiltIn.digit);
368     digits.setMaxCount(Pattern.UNBOUNDED);
369     Pattern dollars=new CharSequencePattern(" dollars");
370     Pattern amount=new ConcatenatePattern(digits,dollars);
371     Matcher matcher=amount.matcher
372        ("I say that 17 dollars + 9 dollars = 26 dollars everyday.");
373     StringBuffer buffer=new StringBuffer();
374     while(matcher.find())
375        matcher.appendReplacement(buffer,"$"+matcher.group(digits));
376     matcher.appendTail(buffer);
377     System.out.println(buffer);
378     *</pre></blockquote>.
379     * @param  sb
380     *         The target string buffer
381     *
382     * @param  replacement
383     *         The  literal replacement string. There are no escape characters.
384     *
385     * @return  This matcher
386     *
387     * @throws  IllegalStateException
388     *          If no match has yet been attempted,
389     *          or if the previous match operation failed
390     *
391     */
392    public Matcher appendReplacement(StringBuffer sb, String replacement)
393    {
394        replacement=java.util.regex.Matcher.quoteReplacement(replacement);
395        matcher.appendReplacement(sb,replacement);
396        return this;
397    }

419     * Replaces every subsequence of the input sequence that matches the
420     * pattern with the given replacement string.
421     *

435     * @return  The string constructed by replacing each matching subsequence
436     *          by the replacement string

439    {

441        return matcher.replaceAll(replacement);
442    }

445     * Replaces the first subsequence of the input sequence that matches the

447     *
448     * <p> This method first resets this matcher.  It then scans the input
449     * sequence looking for a match of the pattern.  Characters that are not

459     * is to be used in further matching operations then it should first be

465     *          subsequence by the replacement string.
